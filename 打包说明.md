# MoeFace — EXE 打包说明（auto-py-to-exe）

## 前置准备

```bash
pip install auto-py-to-exe
```

---

## 打包步骤

### 第一步：启动 auto-py-to-exe

```bash
auto-py-to-exe
```

### 第二步：填写配置

| 设置项 | 值 |
|--------|-----|
| Script Location | `recognize.py` |
| Onefile | **One Directory**（推荐，稳定性更好） |
| Console Window | **Window Based**（无黑框） |
| Icon（可选） | 自行准备 `.ico` 文件 |

### 第三步：添加额外文件（重要！）

在 **"Additional Files"** 区域添加以下资源文件，
格式：`源文件/文件夹` → `目标目录`（目标目录填写相对路径）

| 源路径 | 目标目录（Destination） |
|--------|------------------------|
| `lbpcascade_animeface.xml` | `.` |
| `simhei.ttf` | `.` |
| `cname/` | `cname` |

> ⚠️ **data/ 和 features/ 不要打包进去！**
> 这两个文件夹会很大，而且用户需要自己放图片、自己建特征库。
> 打包完成后把 `data/` 和 `features/` 手动复制到 exe 同目录即可。

### 第四步：点击 CONVERT .PY TO .EXE

等待完成，输出目录默认在 `output/` 下。

## 打包后的目录结构

```
output/recognize/
├── recognize.exe          ← 双击运行
├── lbpcascade_animeface.xml
├── simhei.ttf
├── cname/
│   └── name.json
├── data/                  ← 手动复制过来
│   ├── 永雏塔菲/
│   ├── 初音未来/
│   └── ...
└── features/              ← 首次运行后自动生成
```

## 常见打包问题

### 1. 运行 exe 报错找不到 `lbpcascade_animeface.xml`
- 原因：资源文件没有正确打包
- 解决：检查 Additional Files 是否添加了 xml 和 ttf 文件

### 2. 拖拽功能不生效
- 原因：`tkinterdnd2` 需要额外的 DLL 支持
- 解决：在 Additional Files 中额外添加：
  - 源路径：找到你的 Python 环境下的 `tkinterdnd2` 文件夹
    （通常在 `...\site-packages\tkinterdnd2\`）
  - 目标目录：`tkinterdnd2`

### 3. exe 启动时 GPU 不可用（只走 CPU）
- 正常现象，PyTorch CUDA 版打包比较复杂，CPU 模式可以正常使用

### 4. exe 体积很大（几百MB）
- 正常，torch 本身就有 1~2GB，打包会把必要的 .dll 打进去
- 可以用 One Directory 模式（非 One File），启动速度会快很多

### 5. 启动黑屏或闪退
- 改用 **One Directory** 而非 One File 模式
- 或者先用 Console Window 模式打包，看启动时的报错信息

## 给用户分发时的建议

1. 将整个 `output/recognize/` 文件夹打成 zip 压缩包
2. 在 zip 里附上 `data/` 文件夹（含预置角色图片）
3. 告诉用户解压后双击 `recognize.exe` 即可
4. **不需要安装 Python**，不需要安装任何依赖！
